内源性检索（INTRA: INTrinsic Retrieval via Attention）

定义

INTRA 是一种将检索和生成统一在同一个 encoder-decoder 模型内部表示的框架。核心洞察是：attention 本身就是一种查询条件化的信息选择与加权机制——因此预训练的 attention-based encoder-decoder 可以直接从自己的内部表示中检索证据，无需外部 retriever。

由 NVIDIA 团队（Elad Hoffer, Yochai Blau, Edan Kinderman, Ron Banner, Daniel Soudry, Boris Ginsburg）于 2026 年 5 月提出。arXiv: 2605.05806。

核心思想

传统 RAG（retrieval-augmented-generation）将检索和生成分为两个独立模块：retriever 从索引中选择文本，generator 再消费这些文本。这导致 retriever-generator representation mismatch（检索器和生成器的表示空间不一致）。

INTRA 回答一个根本问题：能否让同一个预训练 encoder-decoder 同时完成检索和生成？

答案是可以——因为 cross-attention 已经天然在做 “查询条件化的证据评分”。INTRA 的关键在于将 decoder 的 cross-attention 权重直接用作检索信号。

技术架构

工作流程（两阶段推理）

  1. 离线编码：用 encoder 将语料库的所有 chunk 预编码为表示向量 K={ki​}i=1M​，存入存储池
  2. 检索阶段（Retrieval Pass）：
    • 输入 query 加上 R 个可训练的 retrieval tokens ρ（~164K 参数）
    • Decoder 的 cross-attention 对预编码 chunk 打分：使用 MaxSim（scaled ColBERT-style late-interaction）聚合 token-level attention scores 为 chunk-level 分数
    • 各层 decoder 的查询状态 qℓ​ 通过可学习的层聚合权重 αℓ​ 组合
    • 选择 Top-n 个 chunk 作为检索结果 SINTRA​
  3. 生成阶段（Generation Pass）：decoder 在选中的 chunk 上下文上生成答案

Reverse-QWK 的意义

标准 Transformer 中每个 decoder 层的 key 需要经过 layer-specific 的 RMSNorm + 线性投影，导致无法复用同一个 encoder 状态池。Reverse-QWK 将投影移到 query 侧：

q~​ℓ​=(qℓ​WK,ℓ⊤​)⊙γK,ℓ​

这使 cross-attention logits 保持等价，同时允许所有层共享 Kˉ，只需构建一个 ANN 索引即可服务所有层和所有 attention head。

实验结果

检索性能（Complete-Evidence Recall）

在四个 Wikipedia QA 基准上对比 9 个 baseline（TF-IDF, BM25, MaxSim, Hybrid RAG, BGE, Qwen3-Embedding 0.6B/4B, Qwen3-Emb-4B + Reranker）：

  • 多跳推理场景（HotPotQA, 2Wiki, MuSiQue）：INTRA 全面最优，尤其在需要组装多条证据的 benchmark 上优势明显
  • 单跳场景（NQ）：优势缩小，因为只需找到一条直接证据
  • 关键发现：INTRA 进行的是全语料库打分（corpus-wide scoring），而非仅在初始候选集内 rerank——可以恢复 S0​ 未包含的证据

Generator Compatibility 实验

当把 INTRA 的检索结果给更强的 decoder-only 模型（Mistral-7B, Phi4, Llama3.1-8B, Qwen3.5-27B 等）生成时，性能可以进一步提升。但共享同一个 decoder 做检索和生成能闭合最大的 performance gap——因为检索到的证据与生成器的 attention pattern 天然对齐。

局限性

  1. 仅适用于 encoder-decoder 架构：依赖 cross-attention，decoder-only 模型无法直接使用
  2. 固定语料池：实验在 ~100M token 的固定 corpus 上，未扩展到开放 web 检索
  3. 单模型族验证：仅在 T5Gemma2 4B 上实现，跨尺度和架构的泛化性待验证
  4. 检索延迟未计入效率计算：TTFT benchmark 排除了 ANN 检索时间
  5. 存储开销：预编码 token 级表示比传统检索索引大得多（1B token × 2560d ≈ 2.5TB）

与 Agent Memory 的关系

INTRA 可以看作 agent-memory 的一种实现方式——它将模型的 encoder 激活直接作为存储记忆，decoder attention 作为记忆读取机制。这与传统的 “检索→注入→生成” 流水线形成对比，暗示 attention 本身可能是一种更自然的记忆访问原语。

开放问题

  • 能否将 INTRA 扩展到 decoder-only 模型（如通过 prefix-tuning 或 prompt-based retrieval tokens）？
  • 动态更新的语料库（增量编码 vs 全量重编码）
  • 多模态扩展（视觉 token 的 intrinsic retrieval）
  • 与 agentic RAG 流程的结合（INTRA 作为单步检索块）
