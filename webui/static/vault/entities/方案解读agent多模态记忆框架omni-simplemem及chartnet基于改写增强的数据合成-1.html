一、Agent多模态memory记忆框架OMNI-SIMPLEMEM

来看第一个思路，想打的点是多模态记忆系统，使Agent能够在长时间跨度内保留、组织和回忆来自文本、图像、音频和视频等多种模态的经验 ，所以核心是设计个框架：《OMNI-SIMPLEMEM: Autoresearch-Guided Discovery of Life-long Multimodal Agent Memory 》，https://arxiv.org/pdf/2604.01007，代码在：https://github.com/aiminglab/OmniMem（暂未同步代码），不过可以重点看设计思路：

这里对架构展开介绍：

1、存储前过滤，先过滤冗余，再统一封装成 MAU，冷热分离存储

这里的设计逻辑是多模态数据（图像/音频/视频）体积巨大，直接存会爆炸 ，长期记忆最怕重复、无意义、冗余信息。传统记忆要么全存（低效），要么全丢（漏信息），所以构建的时候做选择性摄入，自动丢弃无用内容，对每种模态用轻量检测器判断“要不要存”。

对于文本 ，计算和近期记忆摘要的Jaccard相似度，超过阈值判定为重复，丢弃；

对于图像/视频帧 ，用CLIP嵌入比较连续帧，场景几乎不变→只存关键帧；音频用VAD语音活动检测，只保留有人声的片段，静音直接扔掉】。

然后，所有通过过滤的内容，统一封装成MAU（MultimodalAtomicUnit） <s,e,p,τ,m,ℓ>

其中：【s：LLM生成的短摘要（10token左右）、e向量嵌入（用于检索）、p：指针→指向冷存储里的原始大文件（图/音/视频）、τ：时间戳、m：模态标记（text/image/audio/video）、ℓ：与其他MAU的结构链接（用于知识图谱）】；

做好之后，工程上采用冷热分离存储 【热存储：存摘要、嵌入、元数据→超快检索；冷存储：存原始大图、音频、视频→省空间；用指针p连接，按需加载】；

2、渐进式混合检索+按层检索+图谱检索

检索是多路检索，并且涉及到知识图谱这一路，所以可以做做知识图谱的构建和link。

自动建实体关系，支持跨会话多跳推理 。用图谱关联所有MAU的实体与关系，让Agent能做跨会话、多跳、关联推理。核心几个点，也可以再看看图谱构建思路：

自动构建知识图谱 【MAU创建时，LLM自动抽取7类实体（人物、地点、物体、事件、概念、时间、组织）+7类关系（located_in、part_of、interacts_with、owns、attended、created_by、related_to）】；

- >实体消解【名称嵌入余弦相似度、Jaro-Winkler字符串相似度，超过阈值就合并为同一个节点】；

- >图谱检索【从查询里提取种子实体->在图谱上做h跳邻域扩展->按距离衰减打分->把高相关MAU并入混合搜索结果】。

除此之外，另外做混合搜索Dense+Sparse，这里的有个技巧，做集合并集（不是分数重排，保留稠密结果的原始排名，把BM25独有的结果追加在后面，这种方式显著优于分数加权融合 ），具体实现上：

稠密检索Dense使用FAISS ，用户查询q→向量嵌入→和MAU嵌入算余弦相似度；稀疏检索Sparse使用BM25，对MAU摘要做关键词匹配；

金字塔检索（PyramidExpansion），在固定token预算下，分三层逐步扩展，不浪费上下文【Level1仅返回摘要（~10token），只给相似度最高的top-k；Level2加载全文/详细描述，相似度超过阈值θ才展开；Level3加载冷存储原始内容，图像/音频/视频，在token预算B内贪心加载 】；

二、基于改写增强的Chart理解数据合成方案ChartNet

来看图表理解数据集合成思路 ，讲的故事是数值幻觉严重、复杂图表泛化差、多步推理能力弱，无法精准提取结构化数据的问题 ，然后现有的数据维度不太够 的问题，所以继续搞数据。

核心思路很直观，以可执行绘图代码为中间表示 ，构建一套自动化、可扩展、高保真的图表生成与增强pipeline，最终形成150万个标准五元组（每个样本均包含可执行绘图代码、渲染图表图像、原始CSV数据表、自然语言摘要、带思维链（CoT）的问答对）、24种（覆盖柱状、折线、散点、直方图、箱线、雷达、树图、多轴图等）图像、6种（Matplotlib、Plotly、Seaborn、Vega-Altair等）绘图库 。

工作报告在《ChartNet: A Million-Scale, High-Quality Multimodal Dataset for Robust Chart Understanding 》，https://arxiv.org/pdf/2603.27064v1，数据地址在：https://huggingface.co/datasets/ibm-granite/ChartNet，在ChartNet上微调的**granite-4.0-3b-vision** 模型在https: //huggingface.co/ibm-granite/granite-4.0-3b-vision。

可以快速看看合成思路，核心就是增强+后续过滤思路 ：

step1、图表转代码重建

这一步以现有图表数据集为种子，通过VLM(pixtral-large-instruct-2411)将种子图像转换为可执行Python绘图代码；

step2、代码引导式增强

这一步利用LLM(gpt-oss-120b 模型)对代码进行迭代改写，更换图表类型、绘图库、数据内容、视觉样式，实现单张种子图批量生成多样化图表【这个是这个方案的核心 】

step3、图表渲染与执行校验

这一步执行生成代码并过滤执行失败的样本，主要用的是渲染引擎，保证代码有效性与图像可渲染性；

step4、自动化质量过滤

这一步由VLM(pixtral-large-instruct-2411)检测文本重叠、标签截断、图例错误、比例失真、语义矛盾等8类视觉缺陷。

step5.多模态属性生成

这一步基于VLM(pixtral-large-instruct-2411)使用代码与图像自动提取CSV数据表、生成自然语言摘要，并基于Vision-R1与LLaVA-CoT 框架生成带完整思维链的问答对，完成五元组构建。
