AI Agent Skill 研究现状、挑战与方向

基于 Wiki 收录的 28+ 篇 Skill Engineering 论文 + 15 篇 Workflow LLM 论文（2023-2026年），综合 Skill Engineering Survey 和 LLM Workflow Survey 两项系统调查总结。PPT 版本：LLM_Skill_综述.pptx。

1. 背景：Skill 成为第一类部署工件

“Agent skills”——打包了过程性知识、可执行资源和触发条件的自包含模块——已从便捷工具演化为 LLM Agent 系统的核心抽象。Xu et al. (2026) 的综述指出，从单体语言模型到模块化、技能装备的 Agent 的转变，标志着 LLM 部署方式的决定性转变。

Skill Engineering 作为一个新兴研究领域，涵盖技能的表示、获取、组织、选择、进化、安全、评估和组合等全生命周期问题。

关键背景事实

  • 学术+工业共振：28+ 篇顶会论文（ICLR/ICML/NeurIPS/NAACL）+ 工业低代码平台（Dify/Coze/n8n）
  • 中国团队主导：ZJU-KG（SkillNet/SkillRouter/SSL）、Alphalab-USTC（Skill1/SkillFlow/SKILL0）、COOLPKU 贡献 60%+ 核心论文

2. 研究现状

2.1 技能表示（Representation）

社区正从非结构化文本技能向结构化、可验证的表示迁移：

方法代表工作核心思想效果
Text-heavySKILL.md, ClawHub自然语言 Markdown 文档人类可读、易编辑
结构化三层解耦SSL (Liang et al., 2026)调度-结构-逻辑三层分离技能发现 +12% MRR，风险评估 +24% F1
可验证工件Metere et al. (2026)双向正确性准则 + 信任模式形式化验证技能行为
跨框架编译SkCC (Ouyang et al., 2026)SKILL.md → 跨框架可移植编译解决框架锁定
约束引导审计Semia (Wen et al., 2026)约束引导的表示合成自动检测安全隐患

趋势：社区正从非结构化文本向结构化、可验证的表示迁移，但尚缺乏统一标准。

2.2 技能获取（Acquisition）—— 三条路径

路径机制代表工作关键数据
A：技能库+检索推理时加载外部技能SkillSynth, EvoSkill, CODE-SHARP3,560 个已验证任务实例
B：RL 内化训练进模型权重SKILL0, Skill1ALFWorld +9.7%，每步 <0.5K tokens
C：终身学习任务序列中积累SkillFlow166 任务/20 族系基准

核心争议：技能应该存储在外部库中（推理时检索）还是内化到权重中（训练时获取）？SKILL0 展示了内化的有效性，Skill1 展示了库方法的可扩展性。尚无定论。

方法代表工作规模

关系图SkillNet200K+ 技能仓库
能力树AgentSkillOS层级 + DAG 编排
图结构组合GraSP关键发现：2-3 个技能的聚焦集合优于全面文档

2.4 技能选择与路由（Selection & Routing）

  • SkillRouter：1.2B 全文检索重排管线，74.0% Hit@1，13× 更少参数，5.8× 更快
  • 核心发现：隐藏技能实现体导致路由准确率下降 31-44pp——全文是关键信号，而非次要元数据
  • SkillRet：大规模技能检索基准

2.5 技能进化（Evolution）

SkillFlow 核心悖论：技能使用率 ≠ 效用

模型技能使用率性能增益
Claude Opus 4.662.65% → 71.08%+8.43 ✅
Kimi K2.566.87%+0.60 ⚠️
Qwen-Coder-Next—退化 ❌

弱模型可以高频使用技能但无法从中受益——存在能力鸿沟。

三种进化范式：

  • SkillFlow：发现→修补→维护（轨迹驱动+评分驱动的技能补丁）
  • SKILL0：内化为进化替代路径（不维护外部库）
  • Skill1：选择+使用+蒸馏协同进化（频率分解信用分配）

2.6 技能安全（Security）— 快速涌现

2026 年 4-5 月间密集出现的子领域：

工作威胁模型核心方法
SkillScope权限滥用细粒度最小权限执行
RouteGuard技能投毒内部信号检测（比间接注入更隐蔽）
ClawGuard工作流劫持EM 侧信道检测（不依赖主机遥测）
Semia配置漏洞约束引导的表示合成审计
Lv et al.跨文件攻击SKILL.md + 脚本 + 参考文档联合审查
Metere et al.信任缺失双向正确性准则 + 人工在环

趋势：从”信任但验证”向**“不信任且强制”**演进。

2.7 技能评估（Evaluation）

SkillNet 的 SCEMA 五维框架（唯一的内在评估体系）：

维度评估方法
SafetyLLM rubric
CompletenessLLM rubric

MaintainabilityLLM rubric

PhD 级人工验证：MAE < 0.03，QWK ≈ 1.000。

2.8 Workflow Generation（相邻方向）— 五条技术路线

Workflow 研究是 Skill 的执行层——关注如何将技能组合成可执行 DAG。详见 llm-workflow-survey-2026。

路线方法代表工作评价
搜索式MCTS 自动搜索最优结构AFlow (ICLR 2025), DSPy无需人工设计，但计算成本高
KG 增强外部知识指导生成Opus (WKG), KnowAgent弥补 LLM 盲区，但 KB 构建成本高
编译式LLM 生成 DAG + 并发调度LLMCompiler (ICML 2024)延迟降低 3-5x
翻译-规划NL→PDDL→PlannerLLM+P, LLM-Modulo可形式化问题有数学保证
微调式大规模真实数据训练WorkflowLLM实际效果最好，但依赖数据

五大共识：

  1. 纯 LLM 规划不可靠——需要外部知识/规划器补充
  2. 评测必须包含可执行性验证（格式正确 ≠ 真能运行）
  3. 并行化是关键——LLMCompiler DAG 调度降低 3-5x 延迟
  4. 不存在银弹——不同技术路线适用不同场景
  5. Workflow（执行层）和 Skill（能力层）正在加速融合

3. 目前存在的问题

3.1 表示碎片化

26+ 个 Agent 框架各自定义技能格式。SKILL.md 正成为事实标准，但仍缺乏：

  • 跨框架的正式语义定义
  • 版本控制和向后兼容机制
  • 技能依赖的声明式管理

3.2 内化 vs 外部库的权衡未解决

维度外部技能库技能内化
更新灵活性高（替换文件即可）低（需重新训练）
上下文成本高（完整技能文本）无
泛化能力限于检索到的技能可能产生新组合
遗忘风险无有（灾难性遗忘）
可审计性高（文件即真相）低（权重不可解释）

技能”蒸馏-检索谱系”尚未被充分研究。

3.3 技能效用鸿沟

Agent 可以高频率使用技能但无法从中受益。问题可能出在：

  • 技能质量评估缺乏标准化度量
  • 选择-执行不匹配：检索到正确技能 ≠ 正确执行
  • 能力依赖：技能效用在很大程度上取决于 Agent 的基线能力

3.4 规模化挑战

  • SkillRouter：80K 技能时路由准确率对全文信号极其敏感
  • GraSP：技能数量与性能非单调——存在”过度技能”问题
  • SkillRet：现有方法在大库上表现不佳

3.5 安全攻击面扩展

技能引入了传统 prompt injection 无法覆盖的新型攻击：

  • 技能投毒（RouteGuard）：恶意指令隐藏在动作导向的技能体中
  • 工作流劫持（ClawGuard）：篡改工具和技能调用顺序
  • 跨文件攻击：SKILL.md + 脚本 + 引用文档的联合利用

3.6 评测基准碎片化

现有基准各自测试技能能力的一个侧面，缺乏统一的 Skill Engineering benchmark suite。

3.7 Workflow 特有的可靠性问题

  • 长程规划中 LLM 幻觉严重（>10 步后正确率急剧下降）
  • 跨平台泛化弱（Apple Shortcuts → Dify → Coze 不互通）
  • 缺乏动态适应（一次生成、全程固定）
  • 可解释性与信任度低

4. 可能的研究方向

4.1 技能表示的形式化语义

  • 统一技能中间表示（IR）：借鉴编译器 IR，定义跨框架通用语义模型
  • 技能类型系统：形式化的输入/输出契约（pre/post-conditions）
  • 语义版本化：类似 SemVer，定义技能变更的兼容性规则

4.2 混合技能架构（库+内化）

  • 频谱式部署：常用技能内化进权重，长尾技能保留在库中
  • 频率驱动蒸馏：使用频率超阈值自动触发内化
  • 反向操作：内化失效时从库重新加载（解决遗忘）
  • 这是当前最具潜力的研究方向之一

4.3 自适应技能选择

  • 上下文感知路由 + Agent 能力剖面 + 历史成功率
  • 能力校准：何时使用技能？（弱 Agent 受益更多还是强 Agent？——数据暗示是强 Agent）
  • 多技能组合优化（GraSP：2-3 个技能最优 → 如何自动确定？）

4.4 技能效用预测与质量治理

  • 标准化质量度量（代码覆盖、测试充分性、安全审计）
  • 技能效用预测：部署前预测对特定 Agent 的实际效用（SkillFlow 发现是关键空白）
  • npm 式注册表治理：安全扫描、质量门控、信任评分

4.5 终身技能进化系统

  • 自动化技能修补（失败轨迹驱动）
  • 技能合并去重 + 负迁移检测
  • 长期技能库维护策略

4.6 技能安全纵深防御

  • 编译时：静态分析和形式化验证
  • 运行时：SkillScope 式最小权限沙箱
  • 监控层：ClawGuard 的 EM 侧信道检测
  • 溯源层：类似供应链安全的 SBOM 技能溯源链

4.7 统一评测框架

  • Skill Engineering Benchmark Suite：覆盖表示、检索、选择、执行、进化、安全
  • 技能-能力对比矩阵：同一技能对不同能力水平 Agent 的效用差异
  • 长期技能维护评测：模拟版本变更和库增长的影响

4.8 Workflow 方向的交叉机会

  • 闭环动态 Workflow：执行感知的重新规划 + 运行时异常分类与恢复
  • 知识增强混合规划：LLM + Symbolic Planner 级联 + 自学习 Action KB
  • 统一 Workflow IR：跨平台（Dify/Coze/n8n/Shortcuts）的中间表示
  • 从代码仓库自动挖掘：CI/CD pipeline → 技能化

4.9 理论深化的机会

结合 Wiki 中 Yi Ma 和 Jiayi Weng 的研究方向，Skill 领域存在以下理论空白：

  • 压缩视角看技能内化：技能内化本质上是将执行过程压缩进权重——这与 closed-loop-transcription 和 rate-reduction 的信息论框架高度一致。「技能压缩的速率-失真 trade-off」是一个未被研究的方向。
  • 启发式系统的技能内核：heuristic-learning 用软件结构替代神经网络参数——技能作为可组合的软件片段，天然适合成为启发式系统的结构单元。
  • Agent Memory 作为技能内化的理论基座：agent-memory 提出的 Forms × Functions × Dynamics 分类中，参数化记忆（Parametric Memory）本质上就是技能内化。

5. 总结性判断

  1. Skill Engineering 正处于学科化初期——有核心问题、活跃社区、初步标杆，但缺乏统一框架和理论基座
  2. 最大的结构性矛盾：外部库的灵活性与权重的零成本推理之间的取舍——混合架构是最有希望的方向
  3. 安全是最快的增长极——2026 年 4-5 月间密集出现 5 篇安全论文，趋势是”从不信任开始，逐步建立信任”
  4. 中国的 SkillNet/SkillRouter/ZJU-KG 群体是主要推动力——28+ 篇论文中中国团队贡献至少 60%
  5. Workflow（执行层）和 Skill（能力层）正在加速融合——统一的 Skill+Workflow 研究范式是下一步的必然方向
  6. 理论深化窗口已打开——压缩理论、启发式学习、Agent Memory 为技能研究提供了深层理论基础，但尚未被充分连接
